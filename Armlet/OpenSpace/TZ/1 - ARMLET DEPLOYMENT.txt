ARMLET DEPLOYMENT
REAL DEPLOY:
=========
MAINSERVER <--- ARMLET SERVER (VM)
на ваш главный сервер нужно будет поставить VmWare. Мы привезем образ виртуальной машины, на которой крутится сервер браслетов. Общение идет по сети через JSON.

ARMLET SERVER (VM)<-- ARMLET GATE(xN)<--DRIVERS<--ARMHUB(xN)
с сервером браслета общаются N ноутбуков в чьи usb воткнуты браслеты-концентраторы. Общение идет между ноутбуками и сервером браслетов идет по WCF. На ноутбуках ставятся пакет драйверов, который создает виртуальный ком-порт поверх USB. На ноутбуке крутится маленький сервис, который принимает и отсылает из ком-порта радиопакеты принимаемые концетратором от браслетов игроков по радио.
Эту часть мы заимствуем с ЦЧ, Крейл знает как это все настроить.

ARMHUB <---- ARMLET <---------- App
Концентратор принимает и отсылает пакеты от браслетов игроков по радио. Эти пакеты обрабатываются приложением. ArmletApi между мной и Крейлом будет садаптировано под ОК.

VIRTUAL DEPLOY:
============
MAINSERVER <--- ARMLET SERVER (VM)
Также. Нам нужно чтобы ваш сервер появился в интернете, чтобы мы могли тестироваться удаленно.

ARMLET SERVER <-- ARMLET GATE <-- VISUALIZER <---App
                           WCF  (VM2) SERIAL   PIPE     ArmletApi
Эта виртуальная схема позволит тестировать логику приложения в эмуляторе браслета на PC.
